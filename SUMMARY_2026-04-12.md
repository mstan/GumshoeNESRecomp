# Gumshoe Investigation Summary — 2026-04-12

## Bug: Any zapper click during gameplay = full reset to title

## Proven Findings

### 1. Dispatch misses actively firing during gameplay

`dispatch_misses.log` after a gameplay session:
```
extra_func 0 0x9BBF   (x5)
extra_func 2 0x9DEA   (x6)
```

**$9BBF (bank 0):** Entity update code. `func_9B91_b0` loads entity state
from `$0602,X`. If active (BNE), branches to `$9BBF` which handles the
entity based on its state (CMP #3, CMP #2, etc.). The recompiler emits
`call_by_address(0x9BBF); return;` but $9BBF has no dispatch entry.
Result: all active entity processing silently skipped.

**$9DEA (bank 2):** Same pattern in `func_9DC4_b2`. BNE past RTS to
code the recompiler treats as a separate function, but has no dispatch entry.

Both are the same recompiler bug: BNE/BEQ that branches past an RTS to
code within the same logical function. The recompiler splits at the RTS
and treats the branch target as a separate function. But it never creates
a dispatch entry for the target address.

### 2. Forced spr0 hack prevents zapper light detection

`runtime.c` line 694-698:
```c
if (g_zapper_trigger && g_zapper_enabled && !(s & 0x40)) {
    g_ppustatus |= 0x40;
    g_spr0_split_active = 1;
    s |= 0x40;  /* return hit immediately */
}
```

When trigger is held, EVERY `$2002` read returns spr0=1. The detection
polling loop at $C658 checks `$2002` before `$4017`. Forced spr0 means
the loop always exits via the spr0 branch, never reaching the `$4017`
light sensor check.

**Proven by `zapper_trace.log`:**
- Per-frame pattern: `$4017=58` (no light, gate blocks it) then `$2002=C0` (forced spr0)
- Phase 2 polling loop never reaches `$4017` check at `$C665`
- Detection always exits with `$CD=0` (no hits)

### 3. $24 is NOT a reset flag

`$24` is set to 1 at frame 0 during RESET and never changes for the
entire session. Only one `STA $24` exists in the ROM (bank 6 at $80E5,
sets it to 0). No `INC $24` or `DEC $24` anywhere. $24=1 is a game mode
flag (likely "zapper connected"), not a reset trigger.

## Detection Code Flow (bank 1, $C5BC-$C6AD)

Ghidra confirms this is ONE function (242 bytes, 107 instructions).
The recompiler splits it into 5+ C functions connected by tail calls.

With $24=1, the detection loop at $C63C always takes the BNE $C649 path
(DEC $D2 directly). Round counter $D2 starts at 1 (from $C6B2), gets
decremented to 0, phase 2 runs once. Forced spr0 causes immediate exit.
$CD remains 0.

When $CD=0: if $E3/$E4 > 0, calls $D2E9 (life penalty). If both 0,
calls $DB36 (sets bit in $05F4).
